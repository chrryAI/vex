## Single Purpose Description

### What is the single purpose of your extension?

Grape's single purpose is to provide **privacy-first app discovery and feedback collection**. The extension enables users to:

1. **Discover curated AI applications** through a privacy-respecting interface
2. **Try recommended apps** without tracking or surveillance
3. **Submit quality feedback** via the integrated Pear feedback system
4. **Earn credits** for constructive contributions

All functionality serves this core purpose: connecting users with quality privacy-first applications while rewarding meaningful feedback.

### How does this extension accomplish its single purpose?

**App Discovery Interface:**

- Badge icon shows available apps count
- Modal displays curated app listings
- Each app includes description, features, and "Try it now" button
- Links open apps in new tabs for user evaluation

**Feedback Collection:**

- "Give Feedback with Pear" button activates feedback mode
- Adds `?pear=true` URL parameter to track feedback sessions
- Displays personalized feedback tips in sidebar
- Submits feedback to AI validation system

**Credit Rewards:**

- AI validates feedback quality (5-20 credits)
- Real-time credit balance updates
- Quota tracking (10 submissions/day)
- Credits unlock premium features

**Privacy Protection:**

- No tracking scripts or analytics
- No data collection or storage
- No cookies or fingerprinting
- All app interactions are user-initiated

### What are the extension's core features?

1. **App Discovery Modal** - Browse curated privacy-first AI apps
2. **Badge Notifications** - Visual indicator of available apps
3. **Pear Feedback System** - Submit and validate user feedback
4. **Credit Management** - Track and display earned credits
5. **Privacy-First Architecture** - Zero tracking or surveillance

## Permissions Justification

### `storage`

**Why needed:** Store user preferences (selected app mode, credit balance, feedback quota)
**How used:** Local storage only - no cloud sync, no data sharing
**User benefit:** Persistent settings across browser sessions

### `activeTab`

**Why needed:** Inject feedback tips sidebar when user clicks "Give Feedback with Pear"
**How used:** Only activates when user explicitly requests feedback mode
**User benefit:** Contextual feedback guidance without leaving the app

### `scripting`

**Why needed:** Enable Pear feedback mode by injecting UI components
**How used:** Adds feedback tips sidebar and URL parameter tracking
**User benefit:** Seamless feedback experience without page reloads

### Host Permissions (`https://grape.chrry.ai/*`, `http://localhost:5173/*`)

**Why needed:** Connect to Grape backend for app listings and feedback validation
**How used:**

- Fetch curated app list
- Submit feedback for AI validation
- Update credit balance
  **User benefit:** Real-time app discovery and instant credit rewards
